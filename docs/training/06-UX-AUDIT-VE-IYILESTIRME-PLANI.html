02 Kritik Bulgular

P0
Form Korkutuculugu — 66 Alan Tek Seferde

ProductForm.js: 579 satir, 66+ form alani, tek Collapse panelinde

Sorun: CNC patronu urun eklemek istiyor. Form acilinca "Parca No, Urun, Aciklama, Birim, Tedarik Tipi, Min Stok, Max Stok, Yeniden Siparis Noktasi, OEM Parca No, Teknik Resim No, Revizyon, Brut Agirlik, Net Agirlik, En, Boy, Yukseklik, Mense, GTIP, Tedarik Suresi, Varsayilan Tedarikci, Alis Fiyati, Satis Fiyati, Para Birimi, Kritiklik Sinifi, Giris KK, Lot Takibi, Seri Takibi, Raf Omru, Depolama Kosullari, Sertifikalar..." gorunce "ben bunu kullanamam" der ve cikar.
Cozum: useFormMode hook'u eklendi ama ProductForm.js'e henuz uygulanmadi. Uygula: ilk 8 alan gorunsun (Parca No, Urun, Birim, Tedarik Tipi, Min Stok, Satis Fiyati, Resim, Not). Alt kisimda "Detayli Bilgi" linki tiklaninca geri kalan 58 alan acilsin.
Simdiki hali
Tek Collapse panelinde 66 alan alt alta. Patron scroll yapmaktan bikiyor. Cogu alan bos kaliyor.
Olmasi gereken
8 temel alan gorunur. "Tum Alanlari Goster ▼" tikla → 4 grup halinde acilir: Teknik Bilgiler, Tedarik, Kalite, Depolama. Her grup kendi Collapse panelinde.
// ProductForm.js'e ekle:
import useFormMode from '@src/hooks/useFormMode'
const { isAdvanced, ToggleButton } = useFormMode('product')
// Her Form.Item'a: style={{ display: shouldShow('fieldName') ? 'block' : 'none' }}
P0
UI Kutuphanesi Karisikligi — Ant Design + Reactstrap + Bootstrap

3 farkli UI framework ayni sayfalarda karisik kullaniliyor

Sorun: Ayni sayfada Reactstrap Card + Ant Design Table + Bootstrap Row/Col + Ant Design Form kullaniliyor. Butonlar bazen Ant <Button>, bazen Reactstrap <Button>. Padding, border-radius, shadow tutarsiz. Kullanici "bu buton neden farkli gorunuyor?" diye dusunuyor.
Cozum (uzun vadeli): Ant Design'i birincil kutupane olarak kabul et. Reactstrap Card → Ant Card'a gecis yap. Kisa vadede: Reactstrap Card'larin icine Ant Design bilesenleri yerlestirilmeye devam etsin ama stil tutarliligi icin global CSS override ekle.
/* Global override — Reactstrap Card'lari Ant Design'a uyumlu hale getir */
.card { border-radius: 12px; border: 1px solid var(--border); }
.card-header { border-bottom: 1px solid var(--border); padding: 12px 16px; }
.card-body { padding: 16px; }
P0
Liste Sayfalarinda Skeleton Loading Yok

Production.js, Sale.js — UILoader spinner gosteriyor

Sorun: Sayfalar yukleniyor ama ekran bembeyaz + ortada donuyor spinner. Patron "bozuldu mu?" diye dusunuyor. Dashboard'da Skeleton var ama liste sayfalarinda yok.
Cozum: UILoader yerine Ant Design <Skeleton> ve <Table skeleton> kullan. Tablo yukleniyor → satir yerine gri placeholder kutuculari gorunsun. Kullanici verinin geldigini gorsel olarak anlar.
Simdiki
Beyaz ekran + mor spinner (UILoader overlay). Kullanici ne kadar bekleyecek bilmiyor.
Olmasi gereken
Tablo satirlari gri placeholder (Skeleton) olarak gorunur. Veri gelince animasyonla dolari. Kullanici "tablo geliyor" diye anlar.
// Ant Design Table skeleton prop'u:
<Table loading={{ spinning: block, indicator: <Skeleton active /> }} ... />
P1
Inline Form Validation Eksik

Cogu form sadece submit'te hata gosteriyor

Sorun: Patron 10 alani doldurdu, "Kaydet" tikliyor, toast ile "Parca No zorunlu!" diyor. Patron formu scroll edip alani aramak zorunda.
Cozum: Ant Design Form.Item rules zaten tanimli. Ama scrollToFirstError ekle ve alan altinda kirmizi mesaj gorunsun (zaten Ant varsaylani — sadece Form'a scrollToFirstError prop'u ekle).
<Form ... scrollToFirstError={{ behavior: 'smooth', block: 'center' }}>
P1
Mobil/Tablet'te Tablo Yatay Scroll Sorunu

Liste sayfalarinda Table'lar responsive degil

Sorun: Operator telefonda uretim listesini aciyor. 10 kolonlu tablo ekrana sigmiyor, yatay scroll lazim ama belli olmuyor. Bazi kolonlar gereksiz yere genis.
Cozum: Ant Table'a scroll={{ x: 900 }} ekle. Mobilde gereksiz kolonlari gizle (responsive: [{breakpoint: 'md', column: {hidden: true}}]). Onemli bilgileri (urun, durum, miktar) her zaman gorunur birak.
<Table scroll={{ x: 900 }} size="middle" ... />
// Kolon: { ..., responsive: ['md'] } // sadece tablet+ gorunur
P1
Dashboard Widget Oncelik Sirasi — CNC Patronu Icin

Home.js: 1352 satir, 10+ widget, hepsi ayni anda gorunuyor

Sorun: Dashboard'da 10+ widget var: uretim, stok, kalite, satis, bakim, makine heatmap... CNC patronuna sadece 3 bilgi lazim: "Bugun kac is var?", "Makineler ne durumda?", "Bu ay kac para?"
Cozum: getDashboardMode() fonksiyonu zaten var. "simple" modda sadece 3 widget goster: productionStatus + machineWorkload + monthlyRevenue. "full" modda tumu. Dashboard'a bu kontrolu ekle.
import { getDashboardMode } from '@src/utility/uiConfig'
const mode = getDashboardMode() // 'simple' or 'full'
// simple: sadece 3 kart | full: 10+ kart
P2
Buton Hiyerarsisi Belirsiz

Bazi sayfalarda 5-6 buton yan yana, hangisi onemli belli degil

Sorun: Teklif formunda: Kaydet, PDF, Gonder, Kopyala, Geri, Sil — 6 buton yan yana. Patron hangisine basacagini bilmiyor.
Cozum: Birincil aksiyon (Kaydet) buyuk + renklı. İkincil aksiyonlar (PDF, Kopyala) kucuk + gri. Tehlikeli aksiyonlar (Sil) ayri + kirmizi + onay gerekli. Maximum 3 gorunen buton, gerisi "..." dropdown'da.
Simdiki
[Kaydet] [PDF] [Gonder] [Kopyala] [Geri] [Sil] — 6 buton yan yana, hepsi ayni boyutta
Olmasi gereken
[← Geri]     [Kaydet ✓] (buyuk, mavi)   [••• Diger] dropdown icinde: PDF, Gonder, Kopyala | [Sil] (kirmizi, en sagda)
P2
Bos Tablo "Aksiyon Onerisi" Yok

AnimatedEmptyState var ama "ne yapmaliyim?" yonlendirmesi yok

Sorun: Patron Uretim sayfasini aciyor, bos. "Veri bulunamadi" yaziyor. Patron ne yapacagini bilmiyor.
Cozum: Bos durumlara aksiyon butonu ekle: "Ilk uretim emrinizi olusturmak icin once bir siparis ekleyin → [Siparislere Git]". Her moduldeki bos duruma o modulun onkosulunu gosteren mesaj + link ekle.
Simdiki
Animasyonlu icon + "Veri bulunamadi" yazisi. Baska bir sey yok.
Olmasi gereken
Animasyonlu icon + "Henuz uretim emri yok" + "Uretim emri olusturmak icin once siparis ekleyin" + [Siparislere Git →] butonu
P2
Tarih/Saat Formatı Tutarsız

Bazen DD/MM/YYYY, bazen YYYY-MM-DD, bazen HH:MM:SS

Sorun: Farkli ekranlarda farkli tarih formati. Patron kafasi karisiyor.
Cozum: Tum tarihler "29 Mart 2026" (insan okunur) veya "29.03.2026" (kisa). Saat lazimsa "29.03.2026 14:30". Constants.js'teki dateFormat'i tum sayfalarda tutarli kullan.
P3
Dark Mode Ant Design Uyumsuzlugu

Dark mode'da bazi Ant Design bilesenlerinin arka plani beyaz kaliyor

Sorun: Dark mode secildiginde bazi dropdown, datepicker, modal'lar beyaz arka planli kaliyor. Goz yoruyor.
Cozum: Ant Design'in ConfigProvider theme prop'unu kullanarak dark token'lari global olarak ayarla. Veya ant-design-dark-theme NPM paketi.
