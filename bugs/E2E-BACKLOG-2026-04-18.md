### BUG-002: Unit Seeding — Auth-Setup'ta Birimler Kaydedilmiyor
**Modül:** Auth-Setup / TenantRegistrationController  
**Gözlem:** "Birimler seed edildi: 0/13" (4 farklı test koşumunda da aynı)  
**Kök neden:** 
1. Auth-setup, tenant register eder → 2 saniye bekler → token alır → Units'e POST atar
2. Bu noktada tenant henüz provisioning'de olabilir → API 202 döner → units kaydedilmez
3. BUG-001 fix'i cache sorununu çözdü ama async job bazen 2s'den uzun sürebilir
**Sonuç:** Yeni tenant'ta birimler boş → Ürün formu birim seçilemiyor  
**Çözüm Önerisi:**
- `auth-setup.spec.js`'de `/register/status` endpoint'ini ACTIVE olana kadar poll et (maks 30s), sonra seed et
- VEYA `TenantSchemaService.CreateSchemaAsync()` içinde 13 standart birimi seed et (tercih edilen)  
**Öncelik:** P0 — Birimsiz tenant'ta Ürün kartı oluşturulamaz

---

### BUG-003: M² ve M³ Birim İsimleri API'de JSON Parse Hatası Veriyor
**Modül:** UnitsController  
**Gözlem:** `{"name":"M²"}` POST isteğine 400 dönüyor  
**Hata:** `"The JSON value could not be converted to System.String. Path: $.name | LineNumber: 0 | BytePositionInLine: 12."`  
**Kök neden:** ² (U+00B2) ve ³ (U+00B3) karakterleri JSON string olarak serileştirilirken ASP.NET Core JSON parser'ı bu karakterleri reddediyor (muhtemelen `AllowTrailingCommas` veya charset ayarı sorunu)  
**Öncelik:** P0 — M² ve M³ alan birimi olarak kullanılamıyor  
**Çözüm:** `appsettings.json` veya `Program.cs`'de JSON serializer options'a `encoder = JavaScriptEncoder.UnsafeRelaxedJsonEscaping` ekle
